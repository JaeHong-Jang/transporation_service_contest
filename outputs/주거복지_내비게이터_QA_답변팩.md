# 주거복지 내비게이터 Q&A 답변팩

## 1. 기존 마이홈, LH청약플러스와 무엇이 다른가요?

기존 서비스는 공식 공고와 신청 기능을 제공하는 핵심 인프라입니다. 저희는 이를 대체하지 않습니다.  
주거복지 내비게이터는 공식 공고 위에 사용자 조건 기반 매칭, 기준별 점수 분해, 공고문 근거 설명, 지도 입지 분석을 얹는 보완 레이어입니다.

## 2. 점수가 당첨 가능성을 의미하나요?

아닙니다. 점수는 당첨 확률이나 입주 가능 확정이 아니라, 사용자 조건과 공고 기준의 부합도를 보여주는 적합도 점수입니다. 최종 자격 판정은 반드시 공식 공고와 접수기관 심사에 따릅니다.

## 3. AI 챗봇이 틀린 답을 하면 어떻게 하나요?

RAG 챗봇은 공고문 원문 근거가 있는 답변만 하도록 설계합니다. 소득 기준, 자산 기준, 모집기간, 제출서류처럼 중요한 항목은 구조화 데이터와 원문 위치를 함께 보여주고, 근거가 없으면 “공고문에서 확인되지 않음”으로 안내합니다.

## 4. 개인정보 리스크는 없나요?

MVP에서는 회원가입 없이 조건 입력 기반으로 1차 매칭합니다. 주민등록번호, 증빙서류, 계좌정보는 받지 않고, 소득·자산은 구간값 또는 최소 입력값으로 처리할 수 있습니다. 저장이 필요한 단계에서는 별도 동의와 암호화, 비식별 처리를 적용합니다.

## 5. 데이터 최신성은 어떻게 보장하나요?

프로토타입 공고 카드에는 출처, 예시 갱신일, 모집 기간, 원문·신청 링크 버튼을 표시합니다. 구현 단계에서는 공고 제목, 기간, 첨부 공고문 해시, 자격 요건의 변경 여부를 감지해 갱신합니다. 발표에서는 “실시간 공식 연동 완료”가 아니라 “공개 데이터 기반 갱신 구조”로 설명합니다.

## 6. 점수화 기준이 임의적이지 않나요?

총점만 제시하지 않고 자격 적합도, 공고 적합도, 입지 적합도, 주거비 부담 완화, 신청 부담, 준비 용이성을 분리해 보여줍니다. 초기에는 공고문 기준과 공개 규칙 기반으로 산정하고, 실제 사용자 테스트를 통해 가중치를 보정합니다.

## 7. 지도 기반 입지 분석은 주관적이지 않나요?

입지는 절대적 우열이 아니라 사용자 생활 조건에 따른 상대적 적합도입니다. 통근 시간, 대중교통, 병원·복지관·학교·공원 같은 생활SOC, 주변 시세를 분리해서 보여주고, 향후 사용자가 중요도를 선택할 수 있게 확장합니다.

## 8. 정책 사각지대 대시보드는 근거가 충분한가요?

정책 결론을 단정하는 화면이 아니라 사각지대 의심 지표를 보여주는 탐색 대시보드입니다. 익명·집계된 조건 매칭 결과를 활용해 “자격 대상은 많지만 현재 공고 연결이 약한 지역”이나 “서류 조건에서 반복적으로 이탈하는 유형”을 찾는 용도입니다.

## 9. 3개월 안에 구현 가능한가요?

기능을 단계로 나눠 구현합니다. 1단계는 공고 수집, 조건 입력, 기본 매칭입니다. 2단계는 기준별 점수화와 설명, 3단계는 지도 분석, 4단계는 RAG 챗봇, 5단계는 정책 대시보드입니다. 발표용 프로토타입은 전체 방향을 보여주되, MVP는 기본 매칭과 근거 설명부터 완성합니다.

## 10. 기대효과를 어떻게 검증하나요?

주거 문제 해결 같은 거대한 효과를 직접 주장하지 않습니다. 대신 공고 탐색 시간 감소, 조건 불일치 공고 클릭 감소, 원문 공고 확인까지의 단계 감소, 상담 전 자가점검 완료율, 신청 전 서류 체크 완료율을 지표로 검증합니다.
